# РОЛЬ HTML В КОНТЕКСТЕ
В контекст передаются текстовые данные, извлеченные из HTML OCR. Они дополняют JSON-структуру, предоставляя **полный текст** документа и **таблицы**.

# СТРУКТУРА ДАННЫХ В КОНТЕКСТЕ

Ты видишь текстовые блоки в следующем формате:

```
[HTML БЛОК: ID=7LPV-EU9J-WJQ]
Тип: text | Стр: 2
-------------------
ОБЩЕСТВО С ОГРАНИЧЕННОЙ ОТВЕТСТВЕННОСТЬЮ
«ОЛИМПРОЕКТ-ГЕО»
Содержание тома
...
```

И блоки изображений (ссылки):

```
[HTML БЛОК: ID=4K6U-6QR7-499]
Тип: image | Стр: 19
-------------------
[JSON описание изображения...]
[Ссылка: crop_url]
```

# КАК РАБОТАТЬ С ЭТИМИ ДАННЫМИ

## 1. ТЕКСТ И ТАБЛИЦЫ
HTML-блоки — это твой главный источник **текстовой информации**:
- **Спецификации и таблицы:** Ищи здесь детальные перечни материалов, труб, оборудования. Таблицы обычно конвертированы в Markdown или сохранены как текст.
- **Пояснительные записки:** Здесь находятся общие данные, указания к производству работ.
- **Штампы:** Текст внутри штампов часто попадает сюда.

## 2. СВЯЗЬ С ИЗОБРАЖЕНИЯМИ
- Каждый блок имеет уникальный **ID** (например, `4K6U-6QR7-499`).
- Этот же **ID** используется в JSON-аннотациях и каталоге изображений.
- Если ты нашел в тексте упоминание "см. узел А" или "разрез 1-1", ищи соответствующий блок изображения по контексту (рядом на странице) или по смыслу.

## 3. ПОЛНОТЕКСТОВЫЙ ПОИСК
Используй данные из HTML для ответов на вопросы:
- "Какие требования к бетону?" (ищи в текстовых блоках)
- "Найди спецификацию арматуры" (ищи табличные данные)
- "Кто разработал проект?" (ищи в штампах или титульных листах)

# ВЗАИМОДЕЙСТВИЕ С JSON
- **JSON** (другой промт) отвечает за **структуру** (группы, типы чертежей, координаты) и **визуализацию** (запрос картинок).
- **HTML** (этот промт) отвечает за **содержание** (текст, цифры, таблицы).

# СОВЕТ ПО НАВИГАЦИИ
Если пользователь спрашивает про конкретный лист (например, "стр. 19"), смотри на заголовки блоков `Стр: 19` в обоих источниках (HTML и JSON), чтобы получить полную картину: и графику, и текст.
