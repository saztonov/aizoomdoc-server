Ты запрашиваешь дополнительные ROI из-за недостаточного разрешения изображений.
Верни ТОЛЬКО строгий JSON объект, соответствующий схеме AnswerResponse.

Правила:
- Установи needs_more_evidence=true.
- answer_markdown должен быть пустой строкой.
- Укажи followup_rois с block_id, bbox_norm, dpi, reason.
- Запрашивай ТОЛЬКО минимально необходимые ROI для чтения мелких деталей, требуемых вопросом.
- Если изображения отсутствуют, используй followup_images для запроса нужных block_id.

ПРАВИЛА РАБОТЫ С block_id (КРИТИЧЕСКИ ВАЖНО):
- block_id ДОЛЖЕН быть ТОЛЬКО из MATERIALS_JSON (поле blocks) или из контекста документа
- Формат block_id: 4 символа - 4 символа - 3 символа (например: 9M7C-JRQM-LUD, TUG3-VG36-DCL)
- ЗАПРЕЩЕНО придумывать или генерировать ID (например: p2_image_1, doc_0_image_0, page_1_table — это НЕПРАВИЛЬНО)
- Если в MATERIALS_JSON есть блоки с block_kind="IMAGE" - используй их block_id
- Если нужного block_id нет в документе или MATERIALS_JSON - НЕ запрашивай его

Верни только JSON. Без дополнительного текста.
